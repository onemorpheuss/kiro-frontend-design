# Frontend Design

Generate distinctive, production-grade frontend interfaces that avoid generic "AI slop"
aesthetics. This power guides you to commit to a bold, intentional aesthetic direction and
implement real, working code with exceptional attention to detail.

The user provides frontend requirements: a component, page, application, or interface to
build. They may include context about purpose, audience, or technical constraints. Your job
is to pick a clear conceptual direction and execute it with precision.

## What this power does

1. **Direct** — Establish purpose, audience, and a single bold aesthetic direction before coding.
2. **Compose** — Apply typography, color, motion, and spatial composition as deliberate choices.
3. **Implement** — Write production-grade, functional code (HTML/CSS/JS, React, Vue, etc.).
4. **Refine** — Polish every detail and verify the result builds, runs, and is accessible.

## Core principles

- **Intentionality over intensity.** Bold maximalism and refined minimalism both work. The
  key is a clear point of view, executed precisely — not how loud the design is.
- **Commit to one direction.** Choose an extreme (brutalist, editorial, luxury, retro-futuristic,
  organic, etc.) and stay true to it across every element.
- **Match complexity to vision.** Maximalist designs need elaborate code and effects.
  Minimalist designs need restraint, precision, and careful spacing/typography.
- **Real, working code.** Ship functional, production-grade implementations, not mockups.
- **Never converge.** Vary themes, fonts, and aesthetics across generations. Avoid defaulting
  to the same "safe" choices every time.

## What to avoid

Never use generic AI-generated aesthetics: overused fonts (Inter, Roboto, Arial, system
fonts), cliched schemes (purple gradients on white), predictable layouts, and cookie-cutter
patterns that lack context-specific character. See `06-avoiding-ai-slop.md` for the full
checklist.
